Sprint 4 Tech - Problem Solving

Kia ora and welcome to my Sprint 4 Tech Blog!

When I was blocked on a simple problem:
I got blocked on a lot of problems this week, but I think the simplest problem I was stuck on was being able to understand how to use different built-in methods.

I tried pseudocoding for those challenges (some I must have deleted on accident) which helped me organise how I would tackle the challenge. I console.logged a lot, and used a lot of help from my peers (I didn't have to ask, as many people had asked already).

I felt pretty unconfident in my abilities, I got stuck on pretty much every challenge. I feel like I missed learning somewhere, but I haven't because I've done everything, it must just be a gap in my understanding of the code.

Even though I struggled I feel like I learned a lot and I am going away from this week feeling good overall. I have reflected on the challenges and when I look at it I think "This code make sense, I get it". I don't think I've fully remembered it but I know I have these completed challenges I can look at if I need to double check how to do something.

When I elegantly solved a problem:
The main problem I had with the "Manipulating JS Objects" challenge was figuring out how to declare an object within an object i.e. terah.children = { ben: { }}. For this problem I mainly did googling and it clicked pretty fast, I think I was only missing the : after ben.

This was one challenge I felt confident in, and I still took things away like the aforementioned way of making an object within an object.

Reflecting on how confident I feel about the below problem-solving techniques/processes.
Pseudocode - Pseudocoding definitely helps me and I am confident is putting down the larger picture in pseudocode. I am not confident with my ability to put down functions/parameters/etc within the pseudocode yet.

Trying something - I am fully confident in my ability to try things. A lot of the time they don't work, but thats the point of trying things.

Rubber ducky method - I haven't encountered a challenge this week where I thought the rubber ducky method would be applicable. I have sort of used it in the past so I am semi-confident I can use it in a helpful way.

Reading error messages - I have been using error messages a lot in the last two weeks, I would say I'm confident in understanding an error message at this point.

Console.log - This week in particular I console.logged a lot of stuff. I definitely am not using it to it's full potential but I am getting confident with it.

Googling - I am a confident googler.

Asking peers for help - I haven't asked for help myself yet, but I have read a lot of the threads on discord and I am confident my peers would help if I did ask myself.

Asking coaches for help - I haven't had to do this yet because any questions I've had have already been answered in #tech-questions.

Improving your process with reflection - This is something I have mainly started doing this week as I was feeling unconfident in most of the challenges. I don't think I quite use this problem-solving process to it's full capability, but I am starting to get into it as things get more commplicated.
